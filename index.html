提升领导力的四重修炼

建立信任

当普通员工变为领导者的时候，一类如鱼得水，另外一类却是觉得还不如普通员工，这是因为思维还停留在执行者的层面，没有建立起“通过别人”的核心

建立团队

学会放手，团队自我进化，而不是时时刻刻盯着团队做事

建立体系

目的是为了当领导者不在的时候团队也能正常工作
如何建立体系？

  1. 制定标准，以麦当劳的文化为例，麦当劳规定面包的厚度、超时多久的食物需要丢掉
  2. 引进技术，标准的制定是简单的，但想要这个标准被执行，技控更能服人

建立文化

建立企业文化能将员工在精神层面达到一致

明确角色地位，避免亲力亲为

管理者是什么？管理者与普通员工的区别在哪里？

  1. 管理者就是通过别人完成任务
    其中“通过别人”这一点比较难，有的管理者将活全派给手下，自己一点也不做，而有的管理者担心员工完成不好这些工作，时时刻刻盯着员工做事或者是自己全部做了
  2. 管理者的职责: 培养员工和打造团队，切记越俎代庖，也切记独揽全活
  3. 如何让手下的人服你？
    想让手下的人服从并不是要”比所有人都强“，在优秀的团队里是无法做到这点的，想要得到员工的尊重和服从，需要
  4. 掌握好大方向
  5. 学会授权，允许员工犯错

管理者的三种角色

大多数中国管理者的日常

  1. 只顾自己做事，没做好协调 ——> 自己的部分完成了，其他人的部分还差很多
  2. 大多数时间在帮忙”善后“ ——> 没时间做自己的工作
  3. “两头不是人” ——> 在员工和上司之间的形象都不好
    出现上述情况的原因是没有明确好管理者的定位

明确管理者的定位

  1. 下层管理，使命必达，无论使用什么手段都要能完成任务
  2. 中层管理，面面俱到，相当于大管家一样，能够传达上级指令、拆分任务、与其他部门协调等等
  3. 高层领导，营造气氛

用游戏的思维让工作变得有趣

  1. 设置明确的团队游戏目标 ——> 例如知衣的愿景就是成为助力服装行业领域的头部玩家
  2. 设置游戏规则 ——> 员工只需要遵循游戏规则，而不用特意地去讨好管理者
  3. 建立及时的反馈 ——> 管理者需要给员工及时的反馈，开会的时候一开始就进入工作内容会让人觉得无趣
  4. 找到自愿加入的人 ——> 找到愿意和公司一起成长的员工

沟通的四大象限

隐私象限

隐私象限就是指自己知道但别人不知道的事情，隐私象限又分为三个层次

  1. DDS【deep dark secret】
    这块指的是人内心深处不愿意为人所知的黑暗部分，人际交往过程中尽可能不要触及到别人的DDS，令人不适
  2. 不好意思说
    例如员工犯了错误但领导不好意思指出该员工的错误，导致员工在下次仍然犯一样的错误，没有改正
  3. 忘了说
    忘记说比”不好意思说“的层次更为严重，例如忘记夸奖表现好的员工、忘记对子女表达爱，却会在员工犯错、子女犯错的时候进行指责，会给人一种”我很差劲“”你对我很不满意“的错觉

盲点象限

通过他人来看到自己的不足之处并加以改进

潜能象限

每个员工都有未被发现的潜能，要善于发现他们的潜能并且帮助他们成长

公开象限

与隐私象限不同，公开象限是自己知道也允许别人知道的部分，公开象限的好处在于增加社会影响度和信任度
如何增大自己的公开象限？

  1. 转换一部分隐私象限，把自己的一些生活经历与他人分享
  2. 转换一部分盲点相信，允许他人指出自己的盲点

学会倾听

  1. 建立情感账户，当对员工进行批评和建议的时候不让对方认为自己是仗着职位训斥
  2. 吸收对方的信息，根据”漏斗理论“，管理者想的100%内容，说出来的只有80，由于文化水平差异导致员工接收的只有60，员工消化过后剩下40，再去执行的时候只有剩下20
  3. 肢体动作比语言更重要，不要让对方误解你是不耐烦

第三章 对象

对象字面量

一个对象字面量就是包含在花括号中的多个键值对，如果属性名是合法的，不强制使用””包裹，如果是非合法的，则必须用””

检索

可以使用 . 或者 [] 的方法从对象中取值

更新

如果属性名已存在则更新，不存在则被扩充到对象中

引用

对象通过引用来传递，不会被复制

原型

每个对象都连接到一个圆形对象，通过字面量创建的对象连接到 Object.prototype
原型连接只有在检索值的时候才会用到，如果对象中不存在某个属性，则到原型对象中去寻找这个属性

反射

检查对象是否具有某种属性

  1. 使用 typeof 判断对象的类型
  2. 使用 hasOwnProperty 方法

枚举

使用 for in 可遍历对象中的属性，缺点是不会按照顺序遍历
如果想要按照顺序遍历，建议使用 for 循环

删除

使用 delete 删除对象中的属性，这种删除不会删除掉原型对象中的属性

第四章 函数

函数对象

函数对象连接到的原型是 Function.prototype ，函数的与众不同在于能被调用

函数字面量

函数字面量包括四个部分

  1. function 保留字
  2. 函数名称，被省略的就是匿名函数
  3. 函数入参 a 和 b
  4. 函数主体

函数调用

在调用函数的时候，除了声明时定义的形参，每个函数还接受两个附加参数 this 和 arguments，参数 this 的值取决于调用模式
调用运算符是跟在函数值表达式后面的圆括号，圆括号内包含0个或者多个表达式，用逗号隔开

方法调用模式

当一个函数被保存为一个对象的属性时，称之为方法，此时的 this 指向对象

函数调用模式

当一个函数并非一个对象的属性时，是被当做一个函数来调用，此时 this 绑定到全局对象

构造器调用模式

一个函数，如果创建的目的就是希望结合 new 前缀来调用，称之为构造函数
如果一个函数前面带上 new 来调用，那么背地里就会连接到该函数的prototype

apply调用模式

apply 接收两个参数，一个是 this 值，另一个是参数数组

arguments

函数被调用的时候，会得到一个 arguments 类数组，这个类数组里面具有参数

返回

一个函数总会有一个返回值，如果没有返回值，则返回 undefined
如果函数调用前面加上了 new 前缀，那么返回值为新创建的对象

异常

使用 try catch 捕捉 throw 抛出的错误

扩充类型的功能

递归

递归函数就是会直接或者间接调用自身的一种函数

什么是尾递归调用？

函数调用自身成为递归，如果一个函数返回自身递归调用的结果就是尾递归

作用域

JavaScript是函数作用域，意味着定义在函数中的参数和变量在函数外部是不可见的

闭包

内部函数比外部函数拥有更长的生命周期

模块

使用函数和闭包构造模块，模块模式的形式是

  1. 定义了私有变量和函数的函数
  2. 利用闭包可以访问私有变量和特权函数的特性
  3. 将特权函数返回，或者是保存

级联

给一些设置或者修改对象的某个状态却没有返回值的方法返回this，实现级联

函数柯里化

就是把函数的多个参数转换为多个单一参数的函数调用

记忆

函数可以将之前的结果存在 memo 数组中，减少不必要的运算
